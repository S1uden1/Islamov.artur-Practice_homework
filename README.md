# Islamov.artur-Practice_homework
Неявный интерфейс – это требования к типу T, которые не выражены явно в коде, но необходимы для корректной работы шаблона:
1. оператор < для сравнения элементов (используется в условии a[i] < b[j]).
2. Поддержка копирующего присваивания.
3. Отсутствие необходимости в конструкторе по умолчанию.

Недостатки:
1. Функция работает только с масссивами, доступными через указатель и размер.
2. Вызывающий код должен гарантировать, что массив c имеет размер не менее sa + sb.
3. Сравнение всегда через operator<, невозможно подменить критерий сортировки.
4. Ручное управление памятью.
5. Всегда выполняется копирование, даже если тип поддерживает более эффективное перемещение.
6. Передан nullptr с ненулевым размером = UB.
